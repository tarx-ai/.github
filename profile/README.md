# TARX

The runtime for personal AI.

TARX is the memory, tools, permissions, evidence, and execution layer for local-first AI. It starts on your Computer and escalates to Supercomputer only when permission and policy allow it.

**TARXAN Inc** · [tarx.com](https://tarx.com)

## What TARX is

TARX is a runtime for work, not a model company, chatbot wrapper, or benchmark chase.

- Computer by default.
- Supercomputer by permission.
- One runtime contract across local, hosted, and private paths.
- Context, memory, route truth, approvals, and evidence.
- MCP-ready tools.
- Proposal-first actions.
- Evidence for routes, tools, approvals, and results.

## Developer path

1. Start with TARX Local (`tarx-cli` / Computer runtime).
2. Use **TARX Desktop** (Mac Beta) for the full local product shell.
3. Build against the runtime contract.
4. Connect local tools and MCP.
5. Inspect evidence.
6. Add hosted or enterprise runtime only when approved.

## Boundaries

- Supercomputer is not enabled by default.
- Actions are proposal-first unless explicitly enabled.
- Secrets belong in protected secrets/Vault flows, not ordinary memory.
- Production voice is not claimed here.
- Windows Desktop is not public yet.
